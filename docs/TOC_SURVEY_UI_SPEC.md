# Standard toC survey UI spec

This document defines the recommended product/UI direction for **consumer-facing survey filling pages** generated by `survey-creator-skill`.

## Goal

Make generated survey pages feel like:
- real toC product forms
- lightweight questionnaire pages
- trustworthy mobile-friendly survey flows

Instead of:
- AI demo pages
- landing-page hero compositions
- design showcase artifacts

## Core design direction

### 1. One main container
Use a single main survey card or a mobile-first full-page container.

### 2. Compact header
Only keep:
- title
- short description
- optional duration / question count / privacy note

### 3. Standardized questions
Every question should look like a real form block, not a decorative design card.

### 4. Restraint over spectacle
Prefer:
- white surfaces
- light gray background
- subtle border
- medium radius
- calm accent color

Avoid:
- hero gradients
- glow
- glassmorphism
- over-layered cards
- showcase-style option tiles

## Recommended visual baseline

- background: `#f5f6f8` or `#f7f8fa`
- main card: `#ffffff`
- primary text: `#1f2329`
- secondary text: `#646a73`
- border: `#e5e6eb`
- card radius: `12px`–`16px`
- input/button radius: `10px`–`12px`

## Component guidance

### Radio / checkbox
- standard row-based selectable items
- clear checked state
- no overdesigned cards by default

### Input
- use stable, standard form controls
- keep focus and error states simple

### Child input
- visually attach to the selected option
- show inline below the option

### Score / NPS
- use compact, evenly sized pills/buttons
- keep visual weight controlled

## Recommended skins

- `consumer-minimal`
- `consumer-polished`
- `consumer-trust`
- `consumer-editorial`
- `consumer-utility`

These should be implemented as **theme/token variations**, not as totally different page structures.

## Product rule of thumb

If a generated survey page looks like a landing page more than a form, it is too “AI-looking” and should be simplified.
