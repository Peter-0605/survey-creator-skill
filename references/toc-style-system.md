# toC style system

This file defines the **generic consumer-facing visual system** for `survey-creator-skill`.

Its purpose is not to copy specific websites.  
Its purpose is to help the skill generate survey pages that:

- feel intentional
- avoid generic AI-looking UI
- remain easy to fill
- stay trustworthy and lightweight
- work on mobile-first consumer channels

This system borrows one important principle from high-quality frontend design workflows:

> choose a clear aesthetic direction first, then execute it consistently

For surveys, that direction must still remain **completion-friendly**.

---

## Core rule

The skill should not output random decorative UI.

Before rendering HTML, decide:
- what the survey is for
- who is filling it
- what emotional tone is appropriate
- how much visual personality the page can carry before hurting completion

Then choose one of the generic toC packs below.

---

## Available generic packs

### `consumer-minimal`
Best for:
- most default toC surveys
- registration forms
- lightweight feedback
- WeChat / in-app / mobile form experiences

Tone:
- quiet
- clean
- form-first
- low cognitive load

Visual direction:
- light neutral background
- white card surface
- subtle border
- very restrained shadow
- calm accent
- straightforward controls

### `consumer-polished`
Best for:
- branded but still practical surveys
- product feedback
- feature feedback
- slightly more premium toC pages

Tone:
- refined
- modern
- productized
- more visual polish, still safe for completion

Visual direction:
- stronger typography hierarchy
- more deliberate spacing
- slightly richer surfaces
- one clearer accent color
- cleaner CTA contrast

### `consumer-trust`
Best for:
- post-purchase feedback
- satisfaction surveys
- hardware / appliance / product quality questionnaires
- cases where reliability matters more than personality

Tone:
- stable
- reassuring
- product-trust oriented

Visual direction:
- clean structure
- practical hierarchy
- reassuring spacing
- strong readability
- crisp bordered sections

### `consumer-editorial`
Best for:
- content-driven consumer surveys
- lifestyle/community/creator topics
- pages that need more atmosphere without becoming a landing page

Tone:
- expressive
- lighter editorial rhythm
- image-friendly
- modern and content-aware

Visual direction:
- stronger title and cover rhythm
- soft section pacing
- slightly warmer or more curated palette
- still keep question answering friction low

### `consumer-utility`
Best for:
- tool-like surveys
- platform feedback
- internal productized toC forms
- users who want compact and efficient UI

Tone:
- functional
- direct
- scan-friendly
- low decoration

Visual direction:
- compact layout
- stronger alignment
- minimal ornament
- efficient option presentation

## Auto-mapping guidance

### Prefer `consumer-minimal` when:
- the user just says toC / consumer style / simple / clean
- the channel is WeChat, mobile web, or in-app
- no stronger visual personality is needed

### Prefer `consumer-polished` when:
- the user wants “更产品化 / 更高级 / 更精致 / 更像正式产品页面”
- the page should feel more branded than a plain form

### Prefer `consumer-trust` when:
- the survey is about purchased products
- the page is post-purchase, after-sales, product satisfaction, or hardware-oriented
- the page should feel dependable more than expressive

### Prefer `consumer-editorial` when:
- the page serves lifestyle, community, content, or creator-facing scenarios
- the user wants more atmosphere or visual personality

### Prefer `consumer-utility` when:
- the user wants “克制 / 干净 / 工具型 / 平台感 / 高效率”
- the page should feel like a polished product utility rather than a campaign page

## Style system rules

### 1. No generic AI aesthetics
Avoid:
- random gradients without purpose
- overlarge rounded corners
- floating glassmorphism shells
- decorative hero sections that interrupt answering
- loud multi-accent palettes

### 2. Typography must be intentional
- title, description, question text, helper text, and action text must each have distinct roles
- do not make everything the same visual weight
- prefer clean hierarchy over decorative headline styling

### 3. One emotional center only
Each page should have one visual center:
- either the survey container
- or the intro block
- or the answer flow

Do not let header, options, and CTA all compete equally.

### 4. Components should look designed, not autogenerated
That means:
- consistent spacing
- coherent border radius
- stable option row rhythm
- aligned control sizes
- selected states that are obvious but not noisy

### 5. Media should feel content-native
Do not show implementation details like:
- `base64`
- `link`
- raw URLs

Render by resource type:
- image → direct image preview
- audio → direct audio player
- video → direct video player
- unsupported → calm fallback message with optional open-resource link

### 6. Mobile-first always wins
The design should work cleanly around:
- 375px
- 390px
- 430px

Desktop can become wider, but should still feel like a natural extension of mobile rather than a separate enterprise layout.

---

## Output rule

The chosen pack is a renderer configuration:
- pass it through `--style-pack`
- do not encode it into schema legality
- changing the style pack must not change payload logic or question legality
